# Full Technical Changelog — WDTT (WireGuard DTLS TURN Tunnel)

Ревизия всех функциональных и системных изменений в версии 1.0.2.

## [v1.0.2] — 03.04.2026

### 🛠️ Сетевое Ядро (Go Client/Server) — Протокол v2
- **Ликвидация Handshake Overhead:** Полностью вырезана синхронная логика подтверждения `GETCONF -> ACK` и `READY -> READY_OK`. Установление туннеля теперь происходит атомарно, без лишних задержек.
- **Повышение живучести сессии:** 
    - Время жизни сессии (TTL) увеличено до **10 минут**. 
    - Исправлена логика удержания канала в условиях агрессивного Android Doze Mode.
- **Отказ от кастомных модулей ядра:** Проект больше не зависит от модуля `xt_fullconnat`. Реализована универсальная логика NAT через стандартные цепочки `iptables/nftables`, что гарантирует бесперебойную работу на любом современном VPS.
- **Поддержка Протокола TCP:** Внедрена полноценная инкапсуляция TCP-трафика внутри DTLS-слоя для работы в сетях с блокировкой UDP.
- **FQDN Addressing:** Добавлен парсинг доменных имен для всех параметров подключения (TURN, Деплой). 


### 📱 Android-клиент (Kotlin)
- **Системные требования:**
    - 📱 **Расширенная совместимость:** Мы расширили поддержку ОС — теперь приложение полноценно работает на устройствах начиная с **Android 10** (ранее требовался Android 11).
    - 📠 **32-Bit Support:** Добавлены сборки нативного ядра для архитектуры `armeabi-v7a` и `x86`, обеспечивая поддержку старых 32-битных устройств.
- **Интеллектуальный ввод данных:**
    - **Интегрирован Парсер Хешей:** Приложение теперь автоматически "выкусывает" чистый хэш из вставленной ссылки на ВК-звонок (удаление мусора, параметров запроса и доменных префиксов).
    - **Очистка Ввода:** Реализовано автоматическое отсеивание лишних пробелов, невидимых символов и переносов строк при вставке данных в поля "IP/Домен" и "Хеши", что исключает 90% ошибок подключения.
    - **Поддержка Доменов (FQDN):** Поля ввода адреса сервера теперь принимают как IP, так и обычные доменные имена (например, `myserver.com`).
- **Масштабируемость:** Каждый добавленный хэш теперь позволяет развернуть доополнительно **24 воркеров** (3 группы по 8 потоков на хеш), обеспечивая суммарную мощность до **72 рабочих потоков**.
- **Redesign «Info Hub»:** 
    - Внедрена система проверки обновлений через GitHub API.
    - Все элементы интерфейса унифицированы.
- **Watchdog v2:** Улучшена логика самоисцеления туннеля при непредвиденном завершении процесса Go-клиента.

### 🚀 Система Развертывания (Deploy)
- **Кастомизация SSH:** Добавлен выбор SSH-порта для деплоя с персистентным сохранением состояния в Jetpack DataStore.
- **Factory Reset:** Скрипт `deploy.sh` теперь автоматически очищает старые конфигурации (`passwords.json`) и логи, что гарантирует "чистый" старт при переустановке.
- **DataStore Sync:** Исправлены коллизии при сохранении паролей и конфигураций сервера в памяти приложения.
